Структура баг-репорта:

Project (Название проекта)
ID (Уникальный номер отчета)
Created Date (Дата создания)
Update Date (Дата обновления)
Title (Заголовок -"Что? Где? При каких условиях?")
Component (Компонент) 
Description (Описание)
Status (Текущий статус)
Steps (Шаги вопроизвдения)
Priority (Приоритет: High, Major, Medium, Low) 
Severity (Важность: Critical, medium, minor)
Incident Type (Тип инцидента: дефект (Bug), запрос на улучшение (Improvement))
Environment (Окружение) 
Actual result (Фактический результат)
Expected Result (Ожидаемый результат)
Assignee (Автор)
Attachments (Прикрепленные файлы)
Workaround (Возможность "обойти баг)
